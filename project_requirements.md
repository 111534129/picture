# 圖片上傳與分享平台功能需求書

## 1. 專案概述
建立一個多元主題的圖片上傳與分享平台，參考「時光小屋」的功能架構，但不侷限於親子主題。平台需支援多種生活記錄（如旅遊、寵物、攝影作品等），並嚴格禁止血腥暴力內容。

## 2. 技術架構
- **後端語言**: Python (建議使用 Flask 或 Django 框架)
- **前端語言**: HTML (搭配 CSS/JS)
- **資料庫**: SQL (如 SQLite, MySQL 或 PostgreSQL)

## 3. 功能列表

### 3.1 使用者系統 (User System)
- **註冊與登入**: 支援 Email 註冊、密碼加密儲存。
- **個人檔案**: 設定頭像、暱稱、個人簡介。
- **權限管理**: 管理員 (Admin) 與一般使用者 (User)。

### 3.2 圖片管理 (Image Management)
- **圖片上傳**: 支援多張圖片上傳、原始畫質保存。
- **相簿分類**: 可建立不同主題的相簿（例如：2023日本遊、我的貓咪）。
- **時光軸 (Timeline)**: 依照拍攝日期或上傳日期自動排序，呈現時光流逝感（參考時光小屋）。
- **智慧標籤 (Optional)**: 依據主題自動或手動標籤。

### 3.3 社交與分享 (Social & Sharing)
- **隱私設定**:
    - 公開 (Public)
    - 僅好友/家人可見 (Private/Shared)
    - 僅自己可見 (Only Me)
- **邀請機制**: 邀請親友加入特定相簿或關注帳號。
- **互動功能**: 按讚、留言評論。

### 3.4 內容安全 (Content Safety)
- **審核機制**: 自動或人工過濾血腥、暴力、色情等違規圖片。
- **檢舉功能**: 使用者可檢舉不當內容。

## 4. 資料庫規劃 (初步 Schema)
- **Users**: `id`, `username`, `email`, `password_hash`, `avatar`
- **Albums**: `id`, `user_id`, `title`, `description`, `privacy_level`
- **Photos**: `id`, `album_id`, `user_id`, `file_path`, `upload_date`, `taken_date`
- **Comments**: `id`, `photo_id`, `user_id`, `content`, `created_at`

## 5. 專案規範
- **主題**: 多元化，不限特定族群。
- **禁忌**: 嚴禁血腥、暴力內容。

## 6. 建議新增功能 (Phase 2: 社群互動與體驗優化)
### 6.1 互動升級
- [ ] **按讚功能 (Likes)**: 讓使用者能對喜歡的相片點展現支持。
- [ ] **通知中心 (Notifications)**: 當有人按讚、留言、追蹤或分享相簿時收到通知。
- [ ] **動態牆 (Activity Feed)**: 匯總好友的最新動態（上傳新相簿、新照片）。

### 6.2 瀏覽體驗
- [ ] **時光軸模式 (Timeline View)**: 這是「時光相簿」的核心，依據拍攝日期 (EXIF) 或上傳時間，以時間軸方式呈現照片回憶。
- [ ] **全站搜尋 (Search)**: 支援搜尋相簿名稱、使用者名稱。
- [ ] **標籤系統 (Tags)**: 允許為照片添加 #標籤，並可點擊標籤查看相關文章。

### 6.3 實用工具
- [ ] **下載原圖 (Download)**: 允許使用者/好友下載原始畫質的照片。
- [ ] **批次管理**: 批次刪除、批次移動照片到其他相簿。
- [ ] **檢舉機制**: 針對不當內容進行檢舉，後台可審核。
